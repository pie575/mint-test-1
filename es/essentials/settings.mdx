---
title: 'Configuración global'
description: 'Controla el aspecto y el comportamiento de tu documentación con docs.json'
icon: 'gear'
---

Cada sitio de Mintlify requiere un archivo `docs.json` con su configuración básica. Consulta las [propiedades](#properties) a continuación para obtener más información.

<div id="properties">
  ## Propiedades
</div>

<ResponseField name="name" type="string" required>
  Nombre de tu proyecto. Se usa para el título global.

  Ejemplo: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos de navegación y sus páginas

  <Expandable title="Navegación">
    <ResponseField name="group" type="string">
      El nombre del grupo.

      Ejemplo: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Las rutas relativas a los archivos MDX que se usarán como páginas.

      Ejemplo: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a una imagen de logotipo o a un objeto con rutas independientes del logotipo para los modos &quot;light&quot; y &quot;dark&quot;

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ruta al logotipo en modo claro
    </ResponseField>

    <ResponseField name="dark" type="string">
      Ruta al logotipo en modo oscuro
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      URL a la que apunta el logotipo al hacer clic
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global

  <Expandable title="Colores">
    <ResponseField name="primary" type="string" required>
      El color principal. Se usa con más frecuencia para el contenido destacado, los encabezados
      de sección y los acentos en modo claro
    </ResponseField>

    <ResponseField name="light" type="string">
      El color principal para el modo oscuro. Se usa con más frecuencia para el contenido destacado,
      los encabezados de sección y los acentos en modo oscuro
    </ResponseField>

    <ResponseField name="dark" type="string">
      El color principal para los botones importantes
    </ResponseField>

    <ResponseField name="background" type="object">
      El color de fondo tanto en modo claro como en modo oscuro

      <Expandable title="Objeto">
        <ResponseField name="light" type="string" required>
          El código de color hexadecimal del fondo en modo claro
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          El código de color hexadecimal del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Un array de enlaces para mostrar en la barra superior

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      El nombre del botón.

      Ejemplo: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      La URL a la que apunta el botón. Ejemplo: `https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Llamada a la acción">
  <Expandable title="Llamada a la acción de la barra superior">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      `link` muestra un botón. `github` muestra información del repositorio, incluido el recuento de estrellas.
    </ResponseField>

    <ResponseField name="url" type="string">
      Si es `link`: a dónde apunta el botón.

      Si es `github`: enlace al repositorio desde el que se cargará la información de GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texto dentro del botón. Solo es obligatorio si `type` es `link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versiones. Usa esto solo si quieres mostrar diferentes versiones
  de la documentación con un menú desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Una lista de anclas con su `icon`, `color` y `url`.

  <Expandable title="Ancla">
    <ResponseField name="icon" type="string">
      El icono de [Font Awesome](https://fontawesome.com/search?q=heart) que se usa para mostrar el ancla.

      Ejemplo: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      El nombre de la etiqueta del ancla.

      Ejemplo: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que indica qué páginas van en el ancla. Por lo general, es el nombre de la carpeta en la que colocas tus páginas.
    </ResponseField>

    <ResponseField name="color" type="string">
      El color hexadecimal del fondo del icono del ancla. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to`, cada una con un color hexadecimal.
    </ResponseField>

    <ResponseField name="version" type="string">
      Se usa si quieres ocultar un ancla hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el ancla hasta que dirijas a alguien directamente a la documentación que contiene.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Anula la configuración predeterminada del ancla superior.

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del ancla superior
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Icono de Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Una lista de pestañas de navegación.

  <Expandable title="Pestañas">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que indica qué páginas van en la pestaña. Por lo general, este
      es el nombre de la carpeta en la que colocas tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración de la API. Obtén más información sobre las páginas de API en [Componentes de API](/es/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los endpoints de la API. Si `baseUrl` es un arreglo, los usuarios pueden alternar entre varias opciones de URL base.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          La estrategia de autenticación que se usa en todos los endpoints de la API.
        </ResponseField>

        <ResponseField name="name" type="string">
          El nombre del parámetro de autenticación que se usa en el playground de API.

          Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          El prefijo predeterminado del campo de autenticación.

          Por ejemplo, si `inputPrefix` es `AuthKey`, el campo de autenticación se completa automáticamente con `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuración del playground de API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Si el playground se muestra, se oculta o solo muestra el endpoint sin interactividad adicional para el usuario (`simple`)

          Obtén más información en las [guías del playground](/es/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Al habilitar esta opción, te aseguras de que el orden de las claves en las páginas de OpenAPI coincida con el orden definido en el archivo OpenAPI.

      <Warning>Este comportamiento pronto se habilitará de forma predeterminada, momento en el que este campo quedará obsoleto.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una URL o ruta relativa que apunta a tu archivo OpenAPI. También puede ser un arreglo para
  varios archivos.

  Ejemplos:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde cada par clave-valor asocia una plataforma con su URL.

  Ejemplo:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      Uno de los siguientes valores: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Ejemplo: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      La URL de la plataforma social.

      Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configura botones de comentarios en las páginas de documentación

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Habilita un botón para que los usuarios puedan sugerir cambios mediante pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Habilita un botón para que los usuarios puedan reportar un problema en la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el selector de modo oscuro.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Fuerza el modo claro u oscuro para los usuarios nuevos. Si no
      se establece, se usa de forma predeterminada la preferencia del sistema operativo del usuario.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Establécelo en true para ocultar el selector de modo claro/oscuro. Puedes combinar `isHidden` con `default` para forzar que tu documentación use solo el modo claro o el modo oscuro. Por ejemplo:

      <CodeGroup>
        ```json Solo modo oscuro
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Solo modo claro
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se muestra detrás de todas las páginas. Consulta ejemplos en
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
