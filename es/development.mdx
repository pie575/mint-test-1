<Info>
  **Requisitos previos**:

  * Node.js versión 19 o superior
  * Un repositorio de documentación con un archivo `docs.json`
</Info>

Sigue estos pasos para instalar y ejecutar Mintlify localmente en tu sistema operativo.

<Steps>
  <Step title="Instala la CLI de Mintlify">
    ```bash
    npm i -g mint
    ```
  </Step>

  <Step title="Vista previa local">
    Ve al directorio que contiene tu archivo `docs.json` y ejecuta:

    ```bash
    mint dev
    ```

    Tendrás una vista previa local de tu documentación en `http://localhost:3000`.
  </Step>
</Steps>

<div id="custom-ports">
  ## Puertos personalizados
</div>

De forma predeterminada, Mintlify usa el puerto 3000. Puedes cambiar el puerto con la opción `--port`. Por ejemplo, para ejecutarlo en el puerto 3333:

```bash
mint dev --port 3333
```

Si el puerto especificado ya está en uso, Mintlify usará automáticamente el siguiente puerto disponible:

```md
El puerto 3000 ya está en uso. Se probará con el 3001.
```

<div id="mintlify-versions">
  ## Versiones de Mintlify
</div>

Cada versión de la CLI está asociada a una versión específica de Mintlify. Si la vista previa local no coincide con la versión de producción, actualiza la CLI:

```bash
npm mint update
```

<div id="validating-links">
  ## Validación de enlaces
</div>

La CLI puede ayudarte a validar los enlaces de tu documentación. Para identificar enlaces rotos, usa el siguiente comando:

```bash
mint broken-links
```

</div>

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Error: No se pudo cargar el módulo &#x22;sharp&#x22; con el runtime darwin-arm64">
    Esto suele deberse a una versión antigua de Node. Prueba lo siguiente:

    1. Elimina la versión de la CLI que tienes instalada actualmente: `npm remove -g mint`
    2. Actualiza a Node v19 o superior.
    3. Vuelve a instalar la CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Problema: Se produce un error desconocido">
    Solución: Ve al directorio raíz de tu equipo y elimina la carpeta `~/.mintlify`. Después, ejecuta `mint dev` de nuevo.
  </Accordion>
</AccordionGroup>

¿Tienes curiosidad por las actualizaciones recientes? Consulta el [registro de cambios de la CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
