<Info>
  **Requisitos previos**:

  * Node.js 19 o superior
  * Un repositorio de documentación con un archivo `docs.json`
</Info>

Sigue estos pasos para instalar y ejecutar Mintlify localmente en tu sistema operativo.

<Steps>
  <Step title="Instala la CLI de Mintlify">
    ```bash
    npm i -g mint
    ```
  </Step>

  <Step title="Vista previa local">
    Navega hasta el directorio de tu documentación donde se encuentra el archivo `docs.json` y ejecuta el siguiente comando:

    ```bash
    mint dev
    ```

    Podrás ver una vista previa local de tu documentación en `http://localhost:3000`.
  </Step>
</Steps>

<div id="custom-ports">
  ## Puertos personalizados
</div>

De forma predeterminada, Mintlify usa el puerto 3000. Puedes personalizar el puerto en el que se ejecuta Mintlify con la marca `--port`. Por ejemplo, para ejecutar Mintlify en el puerto 3333, usa este comando:

```bash
mint dev --port 3333
```

Si intentas ejecutar Mintlify en un puerto que ya está en uso, usará el siguiente puerto disponible:

```md
El puerto 3000 ya está en uso. Se probará con el 3001.
```

<div id="mintlify-versions">
  ## Versiones de Mintlify
</div>

Cada versión de la CLI está asociada a una versión específica de Mintlify. Si la vista previa local no coincide con la versión de producción, actualiza la CLI:

```bash
npm mint update
```

<div id="validating-links">
  ## Validar enlaces
</div>

La CLI puede ayudarte a validar los enlaces de tu documentación. Para identificar enlaces rotos, usa el siguiente comando:

```bash
mint broken-links
```

  ## Formato del código
</div>

Te sugerimos usar extensiones en tu IDE para reconocer y dar formato a MDX. Si usas VSCode, considera la [extensión de MDX para VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para el resaltado de sintaxis y [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para dar formato al código.

<div id="troubleshooting">
  ## Solución de problemas
</div>

<AccordionGroup>
  <Accordion title="Error: No se pudo cargar el módulo &#x22;sharp&#x22; con el entorno de ejecución darwin-arm64">
    Esto puede deberse a una versión desactualizada de Node. Prueba lo siguiente:

    1. Elimina la versión de la CLI instalada actualmente: `npm remove -g mint`
    2. Actualiza a Node v19 o una versión posterior.
    3. Vuelve a instalar la CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Problema: Aparece un error desconocido">
    Solución: Ve a la raíz de tu sistema y elimina la carpeta `~/.mintlify`. Luego, ejecuta `mint dev` de nuevo.
  </Accordion>
</AccordionGroup>

¿Quieres saber qué cambió en la última versión de la CLI? Consulta el [registro de cambios de la CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
