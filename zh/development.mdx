<Info>
  **前提条件**:

  * Node.js 19 或更高版本
  * 包含 `docs.json` 文件的文档仓库
</Info>

按照以下步骤，在你的操作系统上本地安装并运行 Mintlify。

<Steps>
  <Step title="安装 Mintlify 命令行界面">
    ```bash
    npm i -g mint
    ```
  </Step>

  <Step title="本地预览">
    进入包含 `docs.json` 文件的目录，然后运行：

    ```bash
    mint dev
    ```

    你的文档将在本地通过 `http://localhost:3000` 提供访问。
  </Step>
</Steps>

<div id="custom-ports">
  ## 自定义端口
</div>

默认情况下，Mintlify 使用 3000 端口。你可以通过 `--port` 参数更改端口。例如，如需在 3333 端口上运行：

```bash
mint dev --port 3333
```

如果指定端口已被占用，Mintlify 会自动改用下一个可用端口：

```md
端口 3000 已被占用。将改用 3001。
```

<div id="mintlify-versions">
  ## Mintlify 版本
</div>

每个命令行界面版本都对应一个特定的 Mintlify 版本。如果你的本地预览与线上环境不一致，请更新命令行界面：

```bash
npm mint update
```

<div id="validating-links">
  ## 验证链接
</div>

命令行界面可以检查文档中的链接。要找出失效链接，请运行：

```bash
mint broken-links
```

<div id="troubleshooting">
  ## 故障排查
</div>

<AccordionGroup>
  <Accordion title="错误：无法在 darwin-arm64 运行时中加载 &#x22;sharp&#x22; 模块">
    这通常是由于 Node 版本过旧导致的。请尝试按以下步骤操作：

    1. 移除当前已安装的命令行界面版本：`npm remove -g mint`
    2. 升级到 Node v19 或更高版本。
    3. 重新安装命令行界面：`npm i -g mint`
  </Accordion>

  <Accordion title="问题：遇到未知错误">
    删除主目录中的 `~/.mintlify` 文件夹，然后再次运行 `mint dev`。
  </Accordion>
</AccordionGroup>

想了解最近有哪些更新？请查看 [命令行界面更新日志](https://www.npmjs.com/package/mintlify?activeTab=versions)。
