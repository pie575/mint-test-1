导航菜单就是你的文档站点中每个页面上的链接列表。

你每次添加新页面时都需要更新 `docs.json`。页面不会自动显示出来。

<div id="navigation-syntax">
  ## 导航语法
</div>

导航语法支持递归，因此你可以创建嵌套的导航分组。页面名称中无需包含 `.mdx`。

<CodeGroup>
  ```json Regular Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json Nested Navigation
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

<div id="folders">
  ## 文件夹
</div>

将 MDX 文件放入文件夹中，并更新 `docs.json` 中的路径。

例如，要让页面显示在 `https://yoursite.com/your-folder/your-page`，你可以创建一个名为 `your-folder` 的文件夹，并在其中放入一个名为 `your-page.mdx` 的 MDX 文件。

<Warning>
  除非将其放在另一个文件夹内，否则不能将文件夹命名为 `api`。Mintlify 使用 Next.js，而 Next.js 会将顶层的 `api` 文件夹保留用于内部服务器调用。像 `api-reference` 这样的文件夹名称则可以使用。
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

<div id="hidden-pages">
  ## 隐藏页面
</div>

未包含在 `docs.json` 中的 MDX 文件不会显示在侧边栏中，但仍可通过搜索栏找到，也可以通过直接链接访问。
