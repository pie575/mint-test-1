导航菜单就是文档站点每个页面上显示的链接列表。

每次添加新页面时，都要更新 `docs.json`——页面不会自动列出。

<div id="navigation-syntax">
  ## 导航语法
</div>

导航语法采用递归结构，这意味着你可以创建嵌套的导航分组。页面名称中无需包含 `.mdx`。

<CodeGroup>
  ```json 常规导航
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json 嵌套导航
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

<div id="folders">
  ## 文件夹
</div>

将 MDX 文件放入文件夹中，并更新 `docs.json` 中的路径。

例如，如果你想让页面显示在 `https://yoursite.com/your-folder/your-page`，请创建一个名为 `your-folder` 的文件夹，并在其中放置一个名为 `your-page.mdx` 的 MDX 文件。

<Warning>
  不能将 `api` 用作顶层文件夹名称。Mintlify 使用 Next.js，而 Next.js 会将 `api` 文件夹保留给内部服务器调用使用。请改用类似 `api-reference` 的名称，或将其嵌套在另一个文件夹内。
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

<div id="hidden-pages">
  ## 隐藏页面
</div>

未包含在 `docs.json` 中的 MDX 文件不会出现在侧边栏中，但仍可通过搜索栏和直接链接访问。
